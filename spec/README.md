# Spec

Canonical, language-agnostic specifications for OpenArmature capabilities, with conformance test fixtures.

Each capability has its own subdirectory:

- `<capability>/spec.md` — the canonical behavioral specification
- `<capability>/conformance/` — language-agnostic test fixtures (YAML inputs + expected outputs, plus markdown
  descriptions)

See [`../GOVERNANCE.md`](../GOVERNANCE.md) for how specs are written, versioned, and changed.
